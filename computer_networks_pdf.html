Computer Networks Complete Guide - Detailed Study Notes

1. INTRODUCTION TO COMPUTER NETWORKS

1.1 What is a Computer Network?

Computer Network: A collection of interconnected computing devices that can communicate and share resources with each other through a communication medium.

Key Components:
  • Nodes: Devices like computers, servers, printers, routers
  • Links: Physical or wireless connections (cables, radio waves)
  • Protocols: Rules for communication (TCP/IP, HTTP, etc.)
  • Network Interface: Hardware/software for network access

1.2 Uses of Computer Networks

  • Resource Sharing:
    • Hardware: Printers, scanners, storage devices
    • Software: Applications, databases, operating systems
    • Internet connection sharing
  • Data Sharing:
    • File transfer between computers
    • Database synchronization
    • Real-time data sharing
  • Communication:
    • Email (SMTP, POP3, IMAP)
    • Instant messaging
    • Video conferencing (VoIP)
    • Social networking
  • Business Applications:
    • E-commerce transactions
    • Online banking
    • Cloud computing services
    • Remote work capabilities

1.3 Network Types (Based on Geographic Coverage)

1. PAN (Personal Area Network)

Definition: Smallest network covering personal workspace (typically 10 meters)
Technologies: Bluetooth, IrDA, Zigbee, USB
Examples: Phone to headphone connection, laptop to mouse

Definition: Network covering small geographic area (building, campus)
Range: Up to few kilometers
Technologies: Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11)
Speed: 10 Mbps to 10 Gbps
Examples: Office network, home Wi-Fi, school computer lab

Definition: Network covering city or metropolitan area
Range: 5-50 kilometers
Technologies: Fiber optic cables, microwave links
Examples: City-wide Wi-Fi, cable TV networks, university campus network

Definition: Network covering large geographic areas (countries, continents)
Range: Unlimited (global)
Technologies: Leased lines, satellites, MPLS, Internet

1.4 Network Devices & Topologies

Network Devices:

Hub (Physical Layer Device):
  • Basic connectivity device with multiple ports
  • Operates at Physical Layer (Layer 1)
  • Broadcasts data to all connected devices
  • Creates single collision domain
  • Half-duplex communication only
  • Largely obsolete due to limitations
Switch (Data Link Layer Device):
  • Intelligent device that learns MAC addresses
  • Operates at Data Link Layer (Layer 2)
  • Maintains MAC address table (CAM table)
  • Creates separate collision domain for each port
  • Supports full-duplex communication
  • Reduces network collisions significantly
How Switch Works:
  1. Learning: Records source MAC addresses from incoming frames
  2. Flooding: Broadcasts frame to all ports if destination unknown
  3. Forwarding: Sends frame to specific port if destination known
  4. Filtering: Drops frame if source and destination on same port
  5. Aging: Removes old MAC entries after timeout (typically 300 seconds)
Router (Network Layer Device):
  • Connects different networks (internetworking)
  • Operates at Network Layer (Layer 3)
  • Makes routing decisions based on IP addresses
  • Maintains routing tables
  • Provides NAT (Network Address Translation)
  • Can filter traffic (basic firewall functionality)

Network Topologies:

Bus Topology:
  • All devices connected to single cable (backbone)
  • Terminators at both ends prevent signal reflection
  • Uses CSMA/CD for media access control
  • Advantages: Simple, cost-effective, easy to extend
  • Disadvantages: Single point of failure, performance degrades with more devices

  • All devices connected to central hub/switch
  • Most common topology in modern LANs
  • Easy to troubleshoot and manage
  • Advantages: No collision issues, easy fault isolation
  • Disadvantages: Central device failure affects entire network

Ring Topology:
  • Devices connected in circular fashion
  • Data travels in one direction (unidirectional)
  • Uses token passing for media access
  • Examples: Token Ring (IEEE 802.5), FDDI

  • Disadvantages: Single break affects entire network
Mesh Topology:
  • Every device connected to every other device
  • Full Mesh: n(n-1)/2 connections for n devices
  • Partial Mesh: Some devices have multiple connections
  • Advantages: High reliability, multiple paths
  • Disadvantages: Expensive, complex to manage

1.5 Internet Basics

Internet: Global network of interconnected networks using TCP/IP protocol suite
Key Components:
  • ISP (Internet Service Provider): Companies providing Internet access
  • Backbone: High-speed data transmission lines
  • Routers: Direct traffic between networks
  • Protocols: Standards for communication (TCP/IP, HTTP, DNS)

Internet Addressing System:

IP Address (Internet Protocol Address):
  • Unique numerical identifier for network devices
  • IPv4: 32-bit address (e.g., 192.168.1.1)
  • IPv6: 128-bit address (e.g., 2001:db8::1)
  • Two parts: Network portion + Host portion
Domain Name System (DNS):
  • Translates human-readable names to IP addresses
  • Hierarchical naming system
  • Components: TLD (.com, .org), SLD (google), subdomain (www)
  • Example: www.google.com → 172.217.164.110
How DNS Resolution Works:
  1. User types URL in browser
  2. Browser checks local DNS cache
  3. Query sent to ISP's DNS server
  4. Recursive queries to Root, TLD, and Authoritative servers
  5. IP address returned to browser
  6. Browser connects to web server using IP address
URL (Uniform Resource Locator):
Format: protocol://domain:port/path?query#fragment
Example: https://www.example.com:443/page?id=123#section1
  • Protocol: http, https, ftp, etc.
  • Domain: www.example.com
  • Port: 443 (optional, default for https)
  • Path: /page
  • Query: ?id=123
  • Fragment: #section1
Packets:
  • Small units of data transmitted over networks
  • Contains header (addressing info) and payload (actual data)
  • Typical size: 1500 bytes (Ethernet MTU)
  • Advantages: Error recovery, load balancing, efficient transmission

2. NETWORK MODELS

2.1 Why Layers?

Layered Architecture Benefits:
  • Modularity: Each layer has specific, well-defined functions
  • Interoperability: Standardized interfaces between layers
  • Maintainability: Changes in one layer don't affect others
  • Troubleshooting: Easier to isolate and fix problems
  • Development: Different teams can work on different layers
  • Reusability: Lower layers can support multiple upper layer protocols
How Layering Works:
  1. Encapsulation: Each layer adds its header to data from above
  2. Transmission: Complete frame sent over physical medium
  3. De-encapsulation: Receiving device removes headers layer by layer
  4. Peer Communication: Each layer communicates with same layer on other device

Layer 4: Application Layer

Function: Provides network services directly to user applications
Protocols:
  • HTTP/HTTPS: Web browsing (ports 80/443)
  • FTP: File transfer (ports 20/21)
  • SMTP: Email sending (port 25)
  • DNS: Domain name resolution (port 53)
  • DHCP: Automatic IP configuration (ports 67/68)
  • Telnet/SSH: Remote terminal access (ports 23/22)

Layer 3: Transport Layer

Function: Provides end-to-end communication services
Key Features:
  • Segmentation and reassembly of data
  • Error detection and correction
  • Flow control
  • Multiplexing using port numbers

Protocols:
  • TCP: Reliable, connection-oriented
  • UDP: Unreliable, connectionless

Layer 2: Internet Layer

Function: Handles routing and logical addressing
Key Features:
  • Logical addressing using IP addresses
  • Routing between different networks
  • Packet forwarding
  • Fragmentation and reassembly

Protocols:
  • IP: Internet Protocol (IPv4/IPv6)
  • ICMP: Internet Control Message Protocol
  • ARP: Address Resolution Protocol

Layer 1: Network Access Layer

Function: Handles physical transmission of data
Key Features:
  • Physical addressing using MAC addresses
  • Frame formatting
  • Error detection
  • Media access control

Technologies:
  • Ethernet: Wired LAN technology
  • Wi-Fi: Wireless LAN technology
  • PPP: Point-to-Point Protocol

3. PHYSICAL LAYER

3.1 Signals

Signal: Physical quantity that varies with time to represent information

Analog Signals

Analog Signal: Continuous signal that varies smoothly over time
Characteristics:
  • Amplitude: Maximum strength of signal
  • Frequency: Number of cycles per second (Hz)
  • Phase: Position of waveform in time
  • Period: Time for one complete cycle (T = 1/f)

Examples: Voice, radio waves, sound waves
Analog Signal Formula:
s(t) = A × sin(2πft + φ)
Where: A = Amplitude, f = Frequency, φ = Phase

Digital Signals

Digital Signal: Discrete signal with finite number of values
Characteristics:
  • Binary: Two levels (0 and 1)
  • Bit Rate: Number of bits per second (bps)
  • Bit Interval: Time duration of one bit
  • Square Wave: Abrupt transitions between levels

Advantages: Noise immunity, easy regeneration, error detection
How Digital Transmission Works:
  1. Encoding: Convert data to binary format
  2. Line Coding: Convert binary to electrical signals
  3. Transmission: Send signals over physical medium
  4. Reception: Receive and decode signals
  5. Regeneration: Restore original signal quality

3.2 Transmission Media

Transmission Media: Physical pathway for electromagnetic signals between transmitter and receiver
Types:
  • Guided Media: Signals confined to physical path
  • Unguided Media: Signals propagate through air/space

Guided Media (Wired)

1. Twisted Pair Cable:
Construction: Two insulated copper wires twisted together
Types:
  • UTP (Unshielded Twisted Pair): No shielding, cheaper
  • STP (Shielded Twisted Pair): Foil/braided shielding, better noise immunity

Categories:
  • Cat 3: 10 Mbps, voice applications
  • Cat 5: 100 Mbps, Fast Ethernet
  • Cat 5e: 1 Gbps, Enhanced Cat 5
  • Cat 6: 10 Gbps (short distance), improved specifications
  • Cat 6a: 10 Gbps (100m), augmented Cat 6
Why Twisted Pairs Work:
  • Twisting: Reduces electromagnetic interference (EMI)
  • Cancellation: External noise affects both wires equally, cancels out
  • Crosstalk Reduction: Different twist rates prevent interference between pairs
2. Coaxial Cable:
Construction: Central conductor, insulation, braided shield, outer jacket
Types:
  • Baseband: Single signal (10Base2, 10Base5)
  • Broadband: Multiple signals (Cable TV, Cable Internet)

Advantages: Higher bandwidth than twisted pair, better shielding
Applications: Cable TV, broadband Internet, Ethernet backbones
3. Fiber Optic Cable:
Construction: Glass/plastic core, cladding, buffer, jacket
Principle: Total Internal Reflection of light
Types:
  • Single-mode: Small core (8-10 μm), laser source, long distance (>40km)
  • Multi-mode: Large core (50-100 μm), LED source, shorter distance (<2km)

Advantages: High bandwidth, long distance, immune to EMI, secure
Disadvantages: Expensive, difficult installation, fragile
How Fiber Optics Work:
  1. Light Generation: LED/Laser converts electrical to optical signals
  2. Core Transmission: Light travels through glass core
  3. Total Internal Reflection: Light bounces off core-cladding boundary
  4. Detection: Photodiode converts optical back to electrical

Coaxial Up to 1 GHz 500m Moderate Good Cable TV, Internet
Fiber Optic Up to 75 THz 40+ km High Excellent WANs, backbones

Unguided Media (Wireless)

1. Radio Waves (3 KHz - 1 GHz):
  • Characteristics: Omnidirectional, penetrate buildings
  • Applications: AM/FM radio, TV broadcasting, cellular
  • Advantages: Long range, no line-of-sight needed
  • Disadvantages: Interference, security issues
2. Microwaves (1 GHz - 300 GHz):
  • Characteristics: Directional, line-of-sight required
  • Applications: Satellite communication, Wi-Fi, Bluetooth
  • Advantages: High bandwidth, focused transmission
  • Disadvantages: Weather sensitive, line-of-sight needed
3. Infrared (300 GHz - 400 THz):
  • Characteristics: Very directional, blocked by obstacles
  • Applications: Remote controls, IrDA, short-range communication
  • Advantages: Secure, no interference with radio
  • Disadvantages: Very short range, line-of-sight required

3.3 Modulation & Multiplexing

Modulation Techniques

Modulation: Process of varying properties of carrier signal to encode information
Purpose:
  • Enable long-distance transmission
  • Reduce antenna size requirements
  • Allow multiple signals to share medium
  • Improve signal quality

Analog Modulation:

  • AM (Amplitude Modulation):
    • Information encoded in amplitude variations
    • Carrier frequency remains constant
    • Susceptible to noise
    • Used in AM radio broadcasting
  • FM (Frequency Modulation):
    • Information encoded in frequency variations
    • Amplitude remains constant
    • Better noise immunity than AM
    • Used in FM radio, TV audio
  • PM (Phase Modulation):
    • Information encoded in phase variations
    • Similar to FM in implementation
    • Used in digital communications

Digital Modulation:

  • ASK (Amplitude Shift Keying):
    • Binary 1 = high amplitude, 0 = low amplitude
    • Simple implementation
    • Susceptible to noise
  • FSK (Frequency Shift Keying):
    • Binary 1 = one frequency, 0 = another frequency
    • Better noise immunity than ASK
    • Used in modems
  • PSK (Phase Shift Keying):
    • Binary 1 = phase shift, 0 = no phase shift
    • Most efficient digital modulation
    • Used in satellite communications
  • QAM (Quadrature Amplitude Modulation):
    • Combines amplitude and phase modulation
    • High data rates possible
    • Used in cable modems, Wi-Fi

Multiplexing Techniques

Multiplexing: Technique to transmit multiple signals simultaneously over a single communication channel
Benefits: Efficient use of bandwidth, cost reduction, simplified infrastructure
FDM (Frequency Division Multiplexing):
  • Different frequency bands assigned to different signals
  • Guard bands prevent interference between channels
  • Used in radio/TV broadcasting, cable TV
  • Analog multiplexing technique
How FDM Works:
  1. Signal Separation: Each input signal modulated to different frequency
  2. Combining: All modulated signals combined into composite signal
  3. Transmission: Composite signal sent over single medium
  4. Demultiplexing: Receiver separates signals using filters
TDM (Time Division Multiplexing):
  • Different time slots assigned to different signals
  • Synchronous TDM: Fixed time slots for each input
  • Statistical TDM: Dynamic allocation based on demand
  • Used in digital telephone systems, T1/E1 lines
How TDM Works:
  1. Sampling: Each input sampled in round-robin fashion
  2. Time Slot Assignment: Each sample assigned specific time slot
  3. Frame Creation: All samples combined into TDM frame
  4. Synchronization: Receiver uses timing to separate signals
WDM (Wavelength Division Multiplexing):
  • Different wavelengths (colors) of light for different signals
  • CWDM: Coarse WDM (wide channel spacing, ~20nm)
  • DWDM: Dense WDM (narrow spacing, ~0.8nm, up to 160 channels)
  • Used in fiber optic communications
  • Dramatically increases fiber capacity
CDM (Code Division Multiplexing):
  • Different spreading codes for different signals
  • All signals share same frequency and time
  • Used in CDMA cellular networks
  • Provides inherent security and privacy
Multiplexing Comparison

FDM: |f1|f2|f3|f4| (Frequency domain)
TDM: |t1|t2|t3|t4| (Time domain)
WDM: |λ1|λ2|λ3|λ4| (Wavelength domain)
CDM: |c1+c2+c3+c4| (Code domain)

4. DATA LINK LAYER

4.1 Purpose and Functions

Data Link Layer: Provides reliable node-to-node delivery of data frames
Main Functions:
  • Framing: Organizing bits into frames
  • Physical Addressing: Adding MAC addresses
  • Flow Control: Managing transmission rate
  • Error Control: Detecting and correcting errors
  • Access Control: Controlling access to shared medium
Frame Structure:
Components of a typical frame:
  • Flag/Delimiter: Marks beginning and end of frame
  • Header: Contains addressing and control information
  • Data: Actual information being transmitted
  • Trailer: Contains error detection code (CRC)

Data Link Layer Sublayers:

LLC (Logical Link Control) - IEEE 802.2:
  • Upper sublayer of Data Link Layer
  • Provides interface to Network Layer
  • Handles flow control and error recovery
  • Protocol-independent services
MAC (Media Access Control):
  • Lower sublayer of Data Link Layer
  • Controls access to transmission medium
  • Handles physical addressing (MAC addresses)
  • Protocol-specific (Ethernet, Wi-Fi, etc.)

4.2 Error Detection & Correction

Types of Errors:
  • Single-bit Error: Only one bit in frame is incorrect
  • Burst Error: Multiple consecutive bits are incorrect
  • Random Error: Errors occur randomly throughout frame

Error Detection Methods:

1. Parity Check:
Simple Parity:
  • Add one parity bit to make total 1s even (even parity) or odd (odd parity)
  • Can detect single-bit errors only
  • Cannot detect even number of errors
  • Simple but limited effectiveness

Two-dimensional Parity:
  • Longitudinal Redundancy Check (LRC)
  • Vertical Redundancy Check (VRC)
  • Can detect and correct some burst errors
How Parity Works:
Example with Even Parity:
  • Data: 1101001 (4 ones - even, parity bit = 0)
  • Transmitted: 11010010
  • If received: 11011010 (5 ones - odd, error detected)
2. CRC (Cyclic Redundancy Check):
  • Most widely used error detection method
  • Based on polynomial arithmetic
  • Can detect all single-bit errors
  • Can detect all double-bit errors
  • Can detect odd number of errors
  • Can detect burst errors up to r bits (r = degree of generator)

Common Polynomials:
  • CRC-8: x^8 + x^2 + x + 1
  • CRC-16: x^16 + x^12 + x^5 + 1
  • CRC-32: x^32 + x^26 + x^23 + ... + 1
How CRC Works:
  1. Sender:
    • Append r zeros to data (r = degree of generator)
    • Divide by generator polynomial using XOR
    • Replace zeros with remainder (CRC)
    • Transmit data + CRC
  2. Receiver:
    • Divide received data by same generator
    • If remainder is zero → No error
    • If remainder is non-zero → Error detected
3. Checksum:
  • Sum of all bytes/words in data segment
  • Simple arithmetic calculation
  • Used in TCP and UDP headers
  • Less reliable than CRC for error detection

Error Correction Methods:

Hamming Code:
  • Can detect up to 2-bit errors and correct 1-bit errors
  • Uses parity bits at positions 2^n (1, 2, 4, 8, 16, ...)
  • Number of parity bits: m ≥ log₂(n+m+1) where n = data bits
  • Hamming distance determines error detection/correction capability
How Hamming Code Works:
Example with 4 data bits (D₄D₃D₂D₁):
  1. Position parity bits: P₁(1), P₂(2), D₁(3), P₃(4), D₂(5), D₃(6), D₄(7)
  2. P₁ checks positions 1,3,5,7 (odd positions)
  3. P₂ checks positions 2,3,6,7
  4. P₃ checks positions 4,5,6,7
  5. Error position = P₃P₂P₁ (binary)
Hamming Distance Formula:
To detect d errors: Hamming distance ≥ d + 1
To correct d errors: Hamming distance ≥ 2d + 1

4.3 Flow Control Mechanisms

Flow Control: Mechanism to control the rate of data transmission between sender and receiver to prevent buffer overflow

1. Stop-and-Wait Protocol

Stop-and-Wait:
  • Sender transmits one frame and waits for acknowledgment
  • Only after receiving ACK, sends next frame
  • Simple but inefficient for long-delay links
  • Utilization = 1/(1 + 2a) where a = tₚ/tₜ
How Stop-and-Wait Works:
  1. Sender transmits frame and starts timer
  2. Receiver receives frame, sends ACK
  3. Sender receives ACK, sends next frame
  4. If timeout occurs, sender retransmits frame
Stop-and-Wait Timeline

Sender → Frame 1 → Receiver
Sender ← ACK ← Receiver
Sender → Frame 2 → Receiver
Sender ← ACK ← Receiver

2. Sliding Window Protocol

Sliding Window:
  • Multiple frames can be outstanding (unacknowledged)
  • Window size determines maximum outstanding frames
  • Better utilization than stop-and-wait
  • Two variants: Go-Back-N and Selective Repeat
Go-Back-N ARQ:
  • Receiver accepts frames in order only
  • Discards out-of-order frames
  • Sender retransmits from error point onwards
  • Window size: 1 ≤ W ≤ 2ⁿ-1 (n = sequence number bits)
Selective Repeat ARQ:
  • Receiver accepts out-of-order frames
  • Stores frames in buffer until gap is filled
  • Sender retransmits only errored frames
  • Window size: W ≤ 2ⁿ⁻¹
Protocol Window Size Buffer Required Complexity Efficiency
Stop-and-Wait 1 1 frame Low Low
Go-Back-N 2ⁿ-1 1 frame Medium Medium
Selective Repeat 2ⁿ⁻¹ W frames High High

4.4 Switching Techniques

Switching: Process of connecting input ports to output ports to establish communication paths
Circuit Switching:
  • Dedicated physical path established for entire conversation
  • Three phases: Setup, Data Transfer, Teardown
  • Resources reserved throughout connection
  • Examples: Traditional telephone networks, ISDN
  • Advantages: Guaranteed bandwidth, low latency
  • Disadvantages: Inefficient resource usage, blocking
Packet Switching:
  • Data divided into packets, routed independently
  • Store-and-forward mechanism
  • No dedicated path, resources shared
  • Examples: Internet, X.25, Frame Relay
  • Advantages: Efficient resource usage, robust
  • Disadvantages: Variable delay, potential packet loss
How Packet Switching Works:
  1. Packetization: Data divided into packets with headers
  2. Routing: Each packet routed independently
  3. Store-and-Forward: Intermediate nodes store packets temporarily
  4. Reassembly: Destination reassembles packets in correct order

5. MAC SUBLAYER

5.1 Multiple Access Protocols

Multiple Access Problem: How to coordinate access to a shared communication medium among multiple stations
Categories:
  • Random Access: No coordination, handle collisions
  • Controlled Access: Stations take turns in orderly fashion
  • Channelization: Medium divided into channels

Random Access Protocols:

1. Pure ALOHA:
  • Transmit whenever data is ready
  • If collision occurs, wait random time and retransmit
  • Maximum throughput: 18.4% (S = G × e^(-2G))
  • Vulnerable time: 2 × frame transmission time
  • Simple but inefficient
2. Slotted ALOHA:
  • Time divided into discrete slots equal to frame time
  • Transmission only at slot boundaries
  • Maximum throughput: 36.8% (S = G × e^(-G))
  • Vulnerable time: 1 × frame transmission time
  • Requires global synchronization
ALOHA Comparison

Pure ALOHA: Collision can occur anytime
|----A----|collision|----B----|

Slotted ALOHA: Collision only at slot boundary
|slot1|slot2|slot3|slot4|
| A | B |collision| C |
3. CSMA (Carrier Sense Multiple Access):
  • Listen before transmit ("Sense before Send")
  • Reduces collision probability
  • Variations based on what to do when medium is busy:
    • 1-persistent: Transmit immediately when medium becomes idle
    • Non-persistent: Wait random time before sensing again
    • p-persistent: Transmit with probability p when medium idle
4. CSMA/CD (Collision Detection):
  • Used in Ethernet (IEEE 802.3)
  • Listen while transmitting to detect collisions
  • If collision detected, abort transmission and send jam signal
  • Use binary exponential backoff for retransmission
  • Minimum frame size = 2 × propagation delay × bandwidth
How CSMA/CD Works:
  1. Carrier Sense: Listen to medium before transmission
  2. Transmit: If medium idle, start transmission
  3. Collision Detection: Monitor for collisions during transmission
  4. Jam Signal: If collision detected, send 32-bit jam pattern
  5. Backoff: Wait random time (binary exponential backoff)
  6. Retry: Attempt retransmission (max 16 attempts)
Binary Exponential Backoff:
After nth collision: Wait random time from {0, 1, 2, ..., 2^min(n,10) - 1} × slot time
Slot time = 512 bit times = 51.2 μs (10 Mbps Ethernet)
5. CSMA/CA (Collision Avoidance):
  • Used in Wi-Fi (IEEE 802.11)
  • Cannot detect collisions in wireless (hidden terminal problem)
  • Use RTS/CTS mechanism to avoid collisions
  • Positive acknowledgment for successful transmission
  • Exponential backoff for retransmission
How CSMA/CA Works:
  1. Carrier Sense: Check if medium is idle
  2. IFS Wait: Wait for Inter-Frame Space period
  3. Backoff: Wait random backoff time if medium was busy
  4. RTS: Send Request to Send (optional)
  5. CTS: Receive Clear to Send (optional)
  6. Data: Transmit data frame
  7. ACK: Receive acknowledgment
RTS/CTS Mechanism

Sender → RTS → Receiver
Sender ← CTS ← Receiver
Sender → DATA → Receiver
Sender ← ACK ← Receiver

Hidden nodes hear CTS, defer transmission

Controlled Access Protocols:

Token Passing:
  • Special frame (token) circulates among stations
  • Only token holder can transmit
  • After transmission, release token to next station
  • Examples: Token Ring (IEEE 802.5), Token Bus
  • Advantages: No collisions, bounded access delay
  • Disadvantages: Token overhead, single point of failure
Polling:
  • Primary station polls secondary stations
  • Each secondary transmits when polled
  • Primary controls all communication
  • Used in point-to-multipoint networks
  • Advantages: No collisions, centralized control
  • Disadvantages: Polling overhead, single point of failure

5.2 Ethernet Standards (IEEE 802.3)

Ethernet: Most widely used LAN technology
Naming Convention: [Speed][Signaling][Medium]
  • Speed: 10, 100, 1000 (Mbps)
  • Signaling: BASE (baseband), BROAD (broadband)

6. NETWORK LAYER

6.1 IP Addressing

IP Address: Unique logical identifier assigned to network interfaces
Purpose:
  • Identify network and host portions
  • Enable routing between different networks
  • Provide hierarchical addressing structure

IPv4 Addressing

IPv4: 32-bit address space (4.3 billion addresses)
Format: Dotted decimal notation (192.168.1.1)
Structure: Network portion + Host portion

Classful Addressing:

Class Range Network Bits Host Bits Networks Hosts per Network
A 1.0.0.0 - 126.255.255.255 8 24 126 16,777,214
B 128.0.0.0 - 191.255.255.255 16 16 16,384 65,534

Special IP Addresses:
  • 0.0.0.0: This network (used in routing)
  • 127.0.0.0/8: Loopback addresses
  • 169.254.0.0/16: Link-local addresses (APIPA)
  • 255.255.255.255: Limited broadcast
Private IP Addresses (RFC 1918):
  • Class A: 10.0.0.0/8 (10.0.0.0 - 10.255.255.255)
  • Class B: 172.16.0.0/12 (172.16.0.0 - 172.31.255.255)
  • Class C: 192.168.0.0/16 (192.168.0.0 - 192.168.255.255)

CIDR (Classless Inter-Domain Routing):

CIDR: Replaces classful addressing with flexible subnet masks
Notation: IP address/prefix length (e.g., 192.168.1.0/24)
Benefits:
  • Efficient use of address space
  • Reduces routing table size
  • Supports variable-length subnet masking (VLSM)
How CIDR Works:
  • /24: First 24 bits are network, last 8 bits are host
  • Subnet Mask: 255.255.255.0 for /24
  • Network Address: Logical AND of IP and subnet mask
  • Broadcast Address: All host bits set to 1
CIDR Calculations:
Number of hosts = 2^(32-prefix) - 2
Number of subnets = 2^(borrowed bits)
Example: 192.168.1.0/26
Hosts per subnet = 2^(32-26) - 2 = 62
Subnets available = 2^2 = 4 (if borrowed 2 bits from /24)

Subnetting:

Subnetting: Dividing a network into smaller subnetworks
Benefits:
  • Better network organization
  • Improved security through segmentation
  • Reduced broadcast traffic
  • Efficient IP address utilization
Subnetting Example:
Given: 192.168.1.0/24, create 4 subnets
  1. Determine bits needed: 2^n ≥ 4, so n = 2 bits
  2. New subnet mask: /24 + 2 = /26 (255.255.255.192)
  3. Subnet increment: 256 - 192 = 64
  4. Subnets:
    • 192.168.1.0/26 (hosts: .1-.62, broadcast: .63)
    • 192.168.1.64/26 (hosts: .65-.126, broadcast: .127)
    • 192.168.1.128/26 (hosts: .129-.190, broadcast: .191)
    • 192.168.1.192/26 (hosts: .193-.254, broadcast: .255)

Supernetting (Route Aggregation):

Supernetting: Combining multiple networks into a larger network
Purpose: Reduce routing table size and simplify routing
Example: Combine 192.168.0.0/24, 192.168.1.0/24, 192.168.2.0/24, 192.168.3.0/24 into 192.168.0.0/22

IPv6 Addressing:

IPv6: 128-bit address space (3.4 × 10^38 addresses)
Format: 8 groups of 4 hexadecimal digits (2001:db8::1)
Features:
  • Larger address space
  • Built-in security (IPSec)
  • Auto-configuration
  • No broadcast (uses multicast)
  • Simplified header format
IPv6 Address Types:
  • Unicast: One-to-one communication
  • Multicast: One-to-many communication (FF00::/8)
  • Anycast: One-to-nearest communication

Special Addresses:
  • ::1/128: Loopback address
  • ::/128: Unspecified address
  • FE80::/10: Link-local addresses
  • FC00::/7: Unique local addresses

6.2 Routing

Routing: Process of determining the best path for packets to travel from source to destination
Key Components:
  • Routing Table: Database of network destinations and next hops
  • Routing Metrics: Criteria for path selection (hop count, bandwidth, delay)
  • Routing Protocols: Algorithms for sharing routing information

Routing Algorithms:

1. Distance Vector Routing:
  • Based on Bellman-Ford algorithm
  • Each router maintains distance to all destinations
  • Periodically shares routing table with neighbors
  • Examples: RIP (Routing Information Protocol)
  • Problems: Count-to-infinity, slow convergence
How Distance Vector Works:
  1. Each router knows distance to directly connected neighbors
  2. Routers exchange distance vectors with neighbors
  3. Update routing table using: D(x,y) = min{D(x,z) + D(z,y)} for all z
  4. Repeat until convergence (no more changes)
RIP (Routing Information Protocol):
  • Uses hop count as metric (max 15 hops)
  • Updates every 30 seconds
  • RIP v1: Classful routing
  • RIP v2: Classless routing, authentication
  • Simple but limited scalability
2. Link State Routing:
  • Based on Dijkstra's shortest path algorithm
  • Each router maintains complete network topology
  • Floods link state information to all routers
  • Examples: OSPF (Open Shortest Path First)
  • Faster convergence than distance vector
How Link State Works:
  1. Discover neighbors: Send Hello packets
  2. Measure link costs: Calculate delay, bandwidth
  3. Build LSA: Create Link State Advertisement
  4. Flood LSAs: Distribute to all routers
  5. Build topology: Create complete network graph
  6. Calculate paths: Use Dijkstra's algorithm
OSPF (Open Shortest Path First):
  • Uses cost as metric (based on bandwidth)
  • Supports hierarchical routing (areas)
  • Fast convergence and loop-free
  • Supports authentication and multicast
  • More complex but highly scalable
Feature Distance Vector (RIP) Link State (OSPF)
Algorithm Bellman-Ford Dijkstra
Information Shared Distance to destinations Link state information
Update Frequency Periodic (30 sec) Event-driven
Convergence Slow Fast
Memory Usage Low High
CPU Usage Low High
Scalability Limited High

6.3 Congestion Control

Congestion: Situation where network demand exceeds available capacity
Effects:
  • Increased packet loss
  • Higher delays
  • Reduced throughput
  • Network instability

Congestion Control Mechanisms:

1. Token Bucket Algorithm:
  • Bucket holds tokens, each representing right to send packet
  • Tokens added at constant rate (r tokens/second)
  • Bucket capacity limits burst size
  • If tokens available, packet sent and token removed
  • Smooths traffic and prevents excessive bursts
How Token Bucket Works:
  1. Initialization: Bucket filled to capacity C
  2. Token Addition: Add tokens at rate r per second
  3. Packet Arrival: Check if tokens available
  4. Transmission: If tokens available, send packet and consume token
  5. Blocking: If no tokens, either drop or buffer packet
2. Leaky Bucket Algorithm:
  • Bucket with fixed capacity and constant output rate
  • Packets enter bucket at variable rate
  • Packets leave bucket at constant rate
  • If bucket full, incoming packets dropped
  • Provides constant output rate regardless of input
Token vs Leaky Bucket

Token Bucket:
[Tokens] → |Bucket| → Output (Variable rate)
(Capacity C)

Leaky Bucket:
Input → |Bucket| → Output (Constant rate)
(Variable) (Capacity) (Fixed rate)
Aspect Token Bucket Leaky Bucket
Output Rate Variable (up to token rate) Constant
Burst Handling Allows bursts up to bucket size Smooths all bursts
Buffer Requirement Minimal Requires buffering
Implementation Simple More complex

TCP Flow Control:

Sliding Window Flow Control:
  • Receiver advertises available buffer space (window size)
  • Sender can transmit up to window size without ACK
  • Window slides as acknowledgments received
  • Prevents buffer overflow at receiver
How TCP Flow Control Works:
  1. Window Advertisement: Receiver tells sender available buffer space
  2. Data Transmission: Sender transmits up to window size
  3. ACK with Window Update: Receiver acknowledges and updates window
  4. Window Sliding: Window slides forward as data acknowledged
  5. Flow Control: If window = 0, sender stops transmitting

TCP Congestion Control:

Congestion Control Mechanisms:
  • Slow Start: Exponential increase of congestion window
  • Congestion Avoidance: Linear increase after threshold
  • Fast Retransmit: Retransmit on 3 duplicate ACKs
  • Fast Recovery: Avoid slow start after fast retransmit
TCP Congestion Control Algorithm:
  1. Slow Start: cwnd = 1, double every RTT until ssthresh
  2. Congestion Avoidance: Increase cwnd by 1/cwnd for each ACK
  3. Timeout: ssthresh = cwnd/2, cwnd = 1, restart slow start
  4. 3 Dup ACKs: ssthresh = cwnd/2, cwnd = ssthresh, fast recovery
TCP Congestion Window:
Slow Start: cwnd = cwnd × 2 (exponential)
Congestion Avoidance: cwnd = cwnd + 1/cwnd (linear)
ssthresh = Slow Start threshold

8.1 DNS (Domain Name System)

DNS: Hierarchical naming system that translates domain names to IP addresses
Purpose: Provide human-readable names for network resources
Structure: Distributed database with hierarchical namespace

DNS Components:
  • Root Servers: 13 logical servers worldwide (a.root-servers.net to m.root-servers.net)
  • TLD Servers: Top-Level Domain servers (.com, .org, .edu, country codes)
  • Authoritative Servers: Store DNS records for specific domains
  • Local DNS Servers: ISP servers that cache DNS responses

  1. Query: User types URL, browser checks local cache
  2. Recursive Query: Query sent to local DNS server
  3. Root Query: Local DNS queries root server for TLD server
  4. TLD Query: Query TLD server for authoritative server
  5. Authoritative Query: Query authoritative server for IP address
  6. Response: IP address returned through chain
  7. Caching: Result cached at each level for future queries

8.2 Email Protocols

Email System Components:
  • MUA (Mail User Agent): Email client (Outlook, Gmail)
  • MTA (Mail Transfer Agent): Email server (sendmail, postfix)
  • MDA (Mail Delivery Agent): Delivers mail to mailbox

SMTP (Simple Mail Transfer Protocol):

SMTP: Protocol for sending emails between servers
Port: 25 (standard), 587 (submission), 465 (deprecated SSL)
Characteristics:
  • Text-based protocol
  • Uses TCP for reliable delivery
  • Push protocol (sender initiates)
  • Can only handle ASCII text (MIME for binary)
SMTP Transaction:
  1. Connection: Client connects to server on port 25
  2. HELO/EHLO: Client identifies itself
  3. MAIL FROM: Sender's email address
  4. RCPT TO: Recipient's email address
  5. DATA: Email content transfer
  6. QUIT: Close connection

POP3 (Post Office Protocol v3):

POP3: Protocol for retrieving emails from server
Port: 110 (standard), 995 (SSL/TLS)
Characteristics:
  • Download-and-delete model
  • Simple stateless protocol
  • Limited server-side functionality
  • Good for single-device access

IMAP (Internet Message Access Protocol):

IMAP: Advanced protocol for email access
Port: 143 (standard), 993 (SSL/TLS)
Characteristics:
  • Server-side email storage
  • Multiple device synchronization
  • Folder management on server
  • Partial message download
  • Server-side search capabilities

8.3 FTP (File Transfer Protocol)

FTP: Protocol for transferring files between systems
Ports: 20 (data), 21 (control)
Characteristics:
  • Uses two separate TCP connections
  • Control connection for commands
  • Data connection for file transfer
  • Supports both ASCII and binary modes
FTP Connection Modes:
  • Active Mode:
    • Client opens control connection to server port 21
    • Server opens data connection to client
    • Problems with firewalls/NAT
  • Passive Mode:
    • Client opens both control and data connections
    • Server provides port number for data connection
    • Firewall-friendly
FTP Session Example:
  1. Connection: Client connects to server port 21
  2. Authentication: USER and PASS commands
  3. Navigation: CWD (change directory), LIST (list files)
  4. Transfer: RETR (download), STOR (upload)
  5. Termination: QUIT command
FTP Security Issues:
  • Passwords sent in plain text
  • Data transmitted unencrypted
  • Solutions: FTPS (FTP over SSL/TLS), SFTP (SSH File Transfer Protocol)

9. RECENT TRENDS

9.1 SD-WAN (Software-Defined Wide Area Network)

SD-WAN: Virtual network architecture that uses software to manage and optimize WAN connections
Key Principles:
  • Centralized control and management
  • Separation of control and data planes
  • Application-aware routing
  • Multiple transport options
How SD-WAN Works:
  1. Centralized Controller: Manages policies and routing decisions
  2. Edge Devices: Deploy at branch locations
  3. Overlay Network: Virtual network over multiple transports
  4. Dynamic Path Selection: Choose best path based on application needs
  5. Security Integration: Built-in firewall and encryption
SD-WAN Benefits:
  • Cost Reduction: Use cheaper Internet connections
  • Improved Performance: Application-aware routing
  • Enhanced Security: Integrated security features
  • Simplified Management: Centralized configuration
  • Agility: Rapid deployment of new sites
Traditional WAN vs SD-WAN

Traditional WAN:
Branch → MPLS → Data Center

SD-WAN:
Branch → Internet/MPLS/4G → SD-WAN Controller → Cloud/Data Center
(Multiple paths, intelligent routing)

9.2 CDN (Content Delivery Network)

CDN: Geographically distributed network of servers that deliver content to users from nearest location
Purpose:
  • Reduce latency and load times
  • Improve user experience
  • Reduce bandwidth costs
  • Increase content availability
How CDN Works:
  1. Content Distribution: Original content cached at multiple edge servers
  2. User Request: User requests content from website
  3. DNS Resolution: DNS redirects to nearest CDN server
  4. Cache Check: CDN server checks if content is cached
  5. Content Delivery: Serve from cache or fetch from origin
  6. Response: Content delivered to user from nearest location
CDN Components:
  • Origin Server: Original source of content
  • Edge Servers: Geographically distributed cache servers
  • PoP (Point of Presence): Data center with edge servers
  • Load Balancers: Distribute requests among servers
  • Management System: Monitor and control CDN infrastructure
CDN Caching Strategies:
  • Static Caching: Cache static content (images, CSS, JS)
  • Dynamic Caching: Cache personalized content with TTL
  • Edge Side Includes: Assemble pages at edge servers
  • Push vs Pull:
    • Push: Content pre-loaded to edge servers
    • Pull: Content cached on first request
CDN Architecture

User (US) → CDN Edge (US) → Origin (EU)
User (EU) → CDN Edge (EU) → Origin (EU)
User (Asia) → CDN Edge (Asia) → Origin (EU)

Reduced latency through geographic proximity
Popular CDN Providers:
  • Cloudflare: Global network with security features
  • AWS CloudFront: Amazon's CDN service
  • Google Cloud CDN: Google's global CDN
  • Azure CDN: Microsoft's content delivery network
  • Akamai: One of the largest CDN providers
CDN Performance Metrics:
  • Cache Hit Ratio: Percentage of requests served from cache
  • Origin Offload: Reduction in origin server load
  • TTFB (Time to First Byte): Response time measure
  • Bandwidth Savings: Reduction in origin bandwidth usage
  • Global Coverage: Number of PoPs and geographic spread
