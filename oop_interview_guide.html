Complete Object-Oriented Programming

Interview Questions & Answers Guide

Basic Level (Concept Foundations)

1. What is Object-Oriented Programming?

OOP is a programming paradigm that organizes code around objects containing data (attributes) and methods (functions). It models real-world entities and promotes code reusability, modularity, and maintainability through concepts like inheritance, encapsulation, polymorphism, and abstraction.

2. What are the four main pillars of OOP? Explain each with examples.

Encapsulation: Bundling data and methods together while hiding internal details (private variables with public getters/setters).

Inheritance: Creating new classes from existing ones (Car extends Vehicle).

Polymorphism: Same interface with different implementations (Animal.makeSound() behaves differently for Dog and Cat).

Abstraction: Hiding complex implementation details and showing only essential features (abstract classes/interfaces).

3. What is a class? What is an object?

A class is a blueprint or template that defines attributes and methods for creating objects. An object is an instance of a class that occupies memory and has actual values for the attributes defined in the class.

4. What is the difference between a class and a structure (C++)?

In C++, the main difference is default access: struct members are public by default, while class members are private by default. Both support constructors, destructors, inheritance, and member functions, but structs are typically used for simple data grouping.

5. What is encapsulation? Why is it important?

Encapsulation bundles data and methods within a class while controlling access through access specifiers. It's important for data security (prevents unauthorized access), data integrity (validates modifications), and maintainability (internal changes don't affect external code).

6. What is abstraction? How is it achieved in Java and C++?

Abstraction hides implementation complexity and shows only essential features. In Java: achieved through abstract classes and interfaces. In C++: achieved through pure virtual functions and abstract classes with virtual methods.

7. What are access specifiers (public, private, protected)?

Public: accessible everywhere.
Private: accessible only within the same class.
Protected: accessible within the same class and its subclasses.

They control visibility and access to class members for encapsulation.

8. What is the difference between compile-time and runtime polymorphism?

Compile-time polymorphism (static binding) is resolved during compilation through method overloading and operator overloading - faster execution.

Runtime polymorphism (dynamic binding) is resolved during execution through method overriding and virtual functions - more flexible but slower.

9. What is inheritance? How is it implemented in Java and C++?

Inheritance allows a class to acquire properties and methods from another class. Java: uses extends keyword for single inheritance. C++: uses colon notation and supports multiple inheritance with public/private/protected access levels.

10. What is a constructor and destructor?

Constructor: Special method called when object is created to initialize its state, has same name as class.

Destructor: Special method called when object is destroyed to clean up resources, preceded by ~ in C++. Java uses garbage collection instead of explicit destructors.

Intermediate Level (Language Features)

1. What is method overloading and method overriding?

Method overloading: Multiple methods with same name but different parameters in the same class (compile-time polymorphism).

Method overriding: Child class provides specific implementation of parent's method with same signature (runtime polymorphism).

2. What is the difference between virtual functions (C++) and method overriding (Java)?

C++ requires explicit virtual keyword and methods are non-virtual by default, supports pure virtual functions.

Java methods are virtual by default, uses @Override annotation for clarity, and doesn't need explicit virtual keyword.

3. What is the difference between an abstract class and an interface in Java?

Abstract class: Can have concrete and abstract methods, instance variables, constructors, single inheritance.

Interface: Only abstract methods (before Java 8), constants only, no constructors, supports multiple inheritance.

4. How is multiple inheritance handled in C++ vs Java?

C++: Supports direct multiple class inheritance but can cause diamond problem, solved using virtual inheritance.

Java: No multiple class inheritance, only multiple interface implementation to avoid complexity and ambiguity.

5. What is a friend function in C++? Is there an equivalent in Java?

Friend function in C++ is a non-member function that can access private/protected members of a class using friend keyword. Java has no direct equivalent but uses package-private access and nested classes for similar functionality.

6. What is this keyword and when do you use it?

this refers to the current object instance. Used to distinguish between instance variables and parameters, call other constructors, return current object for method chaining, and pass current object to other methods.

7. What is super keyword in Java and how is it different from base keyword in C++?

Java's super refers to immediate parent class for accessing parent methods, variables, and constructors. C++ has no base keyword; instead uses direct class name or scope resolution operator (::) to access parent class members.

8. What is operator overloading (C++)? Why doesn't Java allow it?

C++ operator overloading allows redefining operators (+, -, ==) for user-defined types making code intuitive. Java doesn't allow it to maintain simplicity, consistency, readability, and prevent confusion about operator behavior.

9. What is a static member (variable or method)?

Static members belong to the class rather than instances, shared among all objects, accessed using class name, loaded when class is loaded. Static methods cannot access instance variables directly.

10. Explain the difference between stack and heap memory allocation.

Stack: Faster, limited size, automatic allocation/deallocation, stores local variables, LIFO structure, thread-local.

Heap: Slower, larger size, manual management (C++) or garbage collected (Java), stores objects, global access, can fragment.

Advanced Level (Design & Deep Concepts)

1. What is a virtual destructor in C++? Why is it important?

Virtual destructor ensures proper cleanup when deleting objects through base class pointers by making destructor calls polymorphic. Without it, only base class destructor is called, leading to memory leaks and incomplete cleanup of derived class resources.

2. What is pure virtual function (C++) and how is it related to abstract classes?

Pure virtual function has no implementation in base class (declared with = 0). It makes the class abstract (cannot be instantiated) and forces derived classes to provide implementation, ensuring consistent interface across inheritance hierarchy.

3. Can we create an object of an abstract class? Why/why not?

No, abstract classes cannot be instantiated because they have incomplete implementation (pure virtual functions). However, you can create pointers/references to abstract classes that point to derived class objects for polymorphic behavior.

4. What is dynamic binding (late binding) vs static binding?

Static binding: Method calls resolved at compile time based on reference type (method overloading, static methods) - faster.

Dynamic binding: Method calls resolved at runtime based on actual object type (method overriding, virtual functions) - flexible but slower.

5. Explain multiple inheritance ambiguity in C++ and how virtual inheritance solves it.

Diamond problem occurs when a class inherits from two classes sharing common base class, causing ambiguity. Virtual inheritance creates single shared instance of base class among all derived classes, eliminating ambiguity and redundancy.

6. How does Java handle multiple inheritance using interfaces?

Java allows single class inheritance but multiple interface implementation. Interfaces provide contract without implementation conflicts. Default method conflicts in interfaces must be explicitly resolved by implementing class.

7. What is a copy constructor in C++? When is it called?

Copy constructor creates new object as copy of existing object with signature ClassName(const ClassName& other). Called during object initialization with another object, passing by value, returning by value, and temporary object creation.

8. What is a deep copy vs shallow copy?

Shallow copy: Copies only immediate members; pointers copied but not pointed data, leading to shared memory and potential double deletion.

Deep copy: Copies all data including dynamically allocated memory, creating independent objects but more expensive.

9. What are inner classes in Java? When do you use them?

Inner classes are classes defined inside another class with access to outer class members including private ones. Used for logical grouping, encapsulation, event handling, and when inner class is only relevant to outer class.

10. How does garbage collection work in Java compared to destructors in C++?

Java uses automatic garbage collection to reclaim memory of unreferenced objects, eliminating manual memory management. C++ requires explicit destructors and manual memory management with new/delete, giving more control but requiring careful resource management.

Real-World & Design Level (Pro Questions)

1. What are design patterns and how do they relate to OOP?

Design patterns are reusable solutions to common programming problems that leverage OOP principles. They provide proven approaches for object interaction, creation, and structure, promoting code reusability, maintainability, and best practices in object-oriented design.

2. Explain the Singleton pattern and how to implement it in Java and C++.

Singleton ensures only one instance of a class exists globally. Implementation involves private constructor, static instance variable, and public static method returning the single instance. Thread safety requires synchronization or eager initialization.

3. What is dependency injection and how does it connect to OOP principles?

Dependency injection provides dependencies to an object rather than the object creating them internally. It promotes loose coupling, testability, and flexibility by removing hard-coded dependencies, aligning with OOP principles of modularity and abstraction.

4. What is the difference between composition and aggregation?

Composition: "has-a" relationship where child cannot exist without parent (strong ownership).

Aggregation: "has-a" relationship where child can exist independently (weak ownership).

Both represent containment but differ in lifecycle dependency.

5. When would you use composition over inheritance?

Use composition when you need "has-a" relationship rather than "is-a," want to avoid inheritance hierarchy complexity, need runtime flexibility, or when inheritance violates Liskov Substitution Principle. Composition provides better encapsulation and flexibility.

6. What is the Liskov Substitution Principle (from SOLID)?

LSP states that objects of derived classes should be substitutable for objects of base classes without breaking application functionality. Derived classes should honor the contract and behavior expected from base class interface.

7. How do you handle object cloning in Java and C++?

Java: Implement Cloneable interface and override clone() method, handle CloneNotSupportedException.

C++: Implement copy constructor and assignment operator for proper object copying, ensuring deep copy for dynamic memory.

8. Explain the difference between interface segregation and abstraction.

Interface segregation: Clients shouldn't depend on interfaces they don't use; prefer multiple specific interfaces over single general one.

Abstraction: Hiding implementation details and showing only essential features through abstract classes/interfaces.

9. What is multiple interface implementation conflict in Java? How do you solve it?

Occurs when implementing multiple interfaces with same default method signatures. Resolved by overriding the conflicting method in implementing class and explicitly calling desired interface method using Interface.super.method().

10. What are some real-life examples where polymorphism is useful?

Media players (different file formats), drawing applications (various shapes), payment processing (different payment methods), database drivers (different databases), and plugin architectures where same interface supports multiple implementations.

Extra Tricky Questions

41. Can a constructor be virtual in C++? Why/why not?

No, constructors cannot be virtual because objects don't exist during construction, so virtual table pointer isn't set yet. Virtual mechanism requires existing object, but constructor's job is to create the object and initialize virtual table pointer.

42. Can we override static methods?

No, static methods belong to class, not instances, so they cannot be overridden. However, they can be hidden in derived classes. Java prevents overriding static methods, while C++ allows hiding with same signature.

43. Why are destructors not needed in Java?

Java uses automatic garbage collection to manage memory, eliminating need for explicit destructors. The garbage collector automatically reclaims memory of unreferenced objects, though finalize() method exists for cleanup but is deprecated.

44. Can enums extend classes or implement interfaces?

In Java, enums implicitly extend java.lang.Enum and cannot extend other classes due to single inheritance, but can implement interfaces. Enums can have constructors, methods, and fields like regular classes.

45. Difference between abstract classes with only abstract methods vs interface?

Abstract class with only abstract methods can have constructors, instance variables, and different access modifiers. Interface cannot have constructors or instance variables, all methods are public by default, and supports multiple inheritance.

Bonus Advanced Questions

46. What is method hiding vs method overriding?

Method hiding: Static methods in subclass hide parent's static method with same signature (compile-time resolution).

Method overriding: Instance methods in subclass override parent's method (runtime resolution with dynamic binding).

47. Explain covariant return types in Java.

Covariant return types allow overriding method to return subtype of original return type. Introduced in Java 5, enables more specific return types in subclasses while maintaining polymorphic behavior and type safety.

48. What is the diamond problem with default methods in Java 8?

When class implements multiple interfaces with same default method, compilation error occurs due to ambiguity. Resolved by explicitly overriding the method and choosing specific interface implementation using Interface.super.method().

49. What is RAII (Resource Acquisition Is Initialization) in C++?

RAII ties resource management to object lifetime - resources acquired in constructor and released in destructor. Ensures automatic cleanup when objects go out of scope, preventing resource leaks even with exceptions.

50. Explain the difference between early binding and late binding with examples.

Early binding: Method calls resolved at compile time (overloaded methods, static methods, private methods).

Late binding: Method calls resolved at runtime (virtual functions, overridden methods) enabling polymorphic behavior.

51. What is slicing problem in C++?

Object slicing occurs when derived object is assigned to base object, losing derived class specific data and behavior. Only base class portion is copied, causing loss of polymorphic behavior and potential data corruption.

52. Can abstract methods be static? Why/why not?

No, abstract methods cannot be static because static methods belong to class and cannot be overridden, while abstract methods must be implemented in subclasses. Static methods are resolved at compile time, abstract methods at runtime.

53. What is the difference between IS-A and HAS-A relationships?

IS-A: Inheritance relationship where subclass is a type of superclass (Dog IS-A Animal).

HAS-A: Composition/Aggregation relationship where class contains another class (Car HAS-A Engine).

Determines whether to use inheritance or composition.

54. Explain method overloading resolution in Java.

Java resolves overloaded methods using most specific match based on: exact match, widening primitive conversions, autoboxing/unboxing, widening reference conversions, and finally varargs. Compilation error if multiple methods match equally.

55. What are phantom references in Java and how do they relate to object cleanup?

Phantom references are weakest references that don't prevent garbage collection but notify when object is about to be finalized. Used for cleanup actions after object becomes unreachable, providing more control than finalize() method.

56. How does virtual inheritance affect object layout in C++?

Virtual inheritance changes object memory layout by adding virtual base table pointer, making base class subobject shared among all derived classes. Increases object size and indirection but eliminates duplication in multiple inheritance scenarios.

57. What is the Curiously Recurring Template Pattern (CRTP) in C++?

CRTP is when class derives from template instantiation of itself. Enables static polymorphism, compile-time method dispatch, and avoiding virtual function overhead while maintaining polymorphic-like behavior through templates.

58. Explain the difference between aggregation and association.

Association: General relationship between classes (Student studies Course).

Aggregation: Special association representing whole-part relationship where parts can exist independently (Department has Employees).

Aggregation is "has-a" relationship with independent lifecycles.

59. What happens if constructor throws exception in Java vs C++?

Java: Object creation fails, memory is automatically cleaned up by garbage collector, partially constructed object is not accessible.

C++: Destructors of already constructed members are called, but object's destructor is not called, requiring careful exception handling.

60. How does name mangling work in C++ and why is it important for function overloading?

Name mangling encodes function signatures into unique symbol names including parameter types, enabling linker to distinguish overloaded functions. Different compilers use different schemes, making binary compatibility challenging but enabling type-safe linking and overloading support.
